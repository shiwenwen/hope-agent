帮我把一个 AI 助理中的上下文迁移到另一个 AI 助理。请回顾我们过去的对话，总结你对用户的了解，并严格按照下方格式输出。

指代规则：
- 禁止使用第一 / 第二人称代词（"我 / 我的 / 你 / 你的"），改用"用户"或中性表述。
- 尽可能保留用户原话，尤其是偏好和明确指令。

分类（填入 `type` 字段）：
- `user`       —— 用户画像事实：姓名、职业、教育、常住地、长期关系
- `feedback`   —— 用户明确要求今后遵循的规则：偏好、禁忌、纠正、语气风格指令
- `project`    —— 带日期的事件、进行中的项目、近期具体计划
- `reference`  —— 用户提到的外部系统、看板、链接、文档位置

输出格式（严格 —— 纯 JSON，不得包含任何其他内容）：
- 只返回一个 JSON 数组，禁止任何说明文字、注释或 Markdown 代码块围栏。
- 每个元素是一个对象，包含以下字段：
  - `content`（字符串，必填）：完整自洽的陈述句，把证据嵌入正文，例如：`证据：用户说"叫我老王"。`
  - `type`（字符串，必填）：必须是 `user` | `feedback` | `project` | `reference` 四个英文值之一 —— 即使 content 本身是中文，type 字段也必须保持英文，因为导入器只识别这四个固定 token。
  - `tags`（1–4 个英文小写短词数组，可选）

示例：
[
  {
    "content": "用户希望被称作"老王"。证据：用户说"叫我老王"。",
    "type": "user",
    "tags": ["name"]
  },
  {
    "content": "用户要求默认使用中文回复。证据：用户说"请始终使用中文回答"。",
    "type": "feedback",
    "tags": ["language", "communication"]
  }
]

要求：
1. 每条记忆都是一句自洽陈述，下游消费者会独立读取每一条。
2. 在 content 中用"证据："前缀嵌入用户原话。
3. `type` 必须是上面四个英文枚举值之一，其他取值会被静默归入 `user`。
4. 如果没有值得记忆的信息，返回 `[]`。
5. 只输出 JSON 数组本身，禁止代码块围栏，禁止任何前后缀文本。
