# 08 · 自主任务

Hope Agent 不只回答问题,还能**持续完成工作**。本章介绍六个配合使用的角色:**目标**定义结果、**工作流**组织执行、**持续推进**决定何时继续、**任务**呈现进度、**执行模式**控制自主程度,以及动手前先规划的**计划模式**。

## 心智模型

这几个角色可以组合使用,也可以单独用。记住它们各自的分工:

| 角色 | 回答的问题 | 命令 |
| --- | --- | --- |
| **目标(Goal)** | 最终要达成什么?怎样算完成?哪些证据算数? | `/goal` |
| **工作流(Workflow)** | 这一次具体怎么执行?(阶段、并行、审阅、验证) | `/workflow` |
| **持续推进(Loop)** | 什么时候再推进一次?(定时、条件、事件) | `/loop` |
| **执行模式(Mode)** | 本会话以多高强度(多主动、多深入)推进? | `/mode` |
| **计划模式(Plan Mode)** | 动手前先把方案写清楚、给我审批 | `/plan` |
| **任务(Task)** | 现在进行到哪一步了? | (自动展示) |

它们边界清晰:目标只管终点和证据、不执行;工作流只管一次执行、不做长期调度;持续推进只管触发、不定义完成标准;执行模式只管强度。

## 8.1 目标(Goal)

给一个长任务设定「最终目标 + 完成标准」,AI 会持续自主推进,完成时给出简短收口和证据。

**怎么用**:

```
/goal 上线官网 v2 --criteria 首页/定价/文档三页可访问且无控制台报错
/goal status      # 查看当前目标状态
/goal pause       # 暂停
/goal resume      # 恢复
/goal evaluate    # 手动评估进度
/goal accept      # 接受当前成果、关闭目标
/goal clear       # 清除目标
```

也可以从输入框的「目标」入口进入,输入框上方会常驻显示当前目标摘要、进度和操作按钮;工作台里有独立的目标区,能展开看每条完成标准的状态、预算、证据和时间线。

**几个要点**:

- 一个会话同一时刻只能有一个进行中的目标。
- **完成不采信 AI 自评**——必须通过一次确定性的「最终审计」(有强证据、无阻塞、所有必须项都满足才算完成)。
- 完成后会等你确认关闭:你可以「接受并关闭」,也可以「要求更严格的证据」把目标退回继续补证据。
- 每一轮结束后,如果目标还需要推进,系统会自动排一个约 10 秒的续跑让 AI 继续,直到完成 / 阻塞 / 暂停 / 预算耗尽。
- 创建目标时可选设 **token / 时间 / 轮次预算**(留空 = 不设限),达到后会拒绝开新的工作流并告警。

> [无痕会话](03-对话与会话.md#36-无痕会话)不能创建目标;有进行中目标的会话也不能转无痕。

## 8.2 工作流(Workflow)

打开工作流模式后,AI 面对复杂任务会**自己写一段执行脚本**,产出一个可观察、可暂停 / 恢复 / 取消、可审批、崩溃能恢复的执行过程。

**怎么用**:

```
/workflow on          # 开启:允许 AI 按需自主编排
/workflow ultracode   # 更强档:实质任务默认考虑多阶段、并行审查、交叉验证(更慢更贵、质量优先)
/workflow off         # 关闭
/workflow status      # 当前模式 + 最近的执行
/workflow trace <id>  # 查看某次执行的步骤
/workflow approve|pause|resume|cancel <id>
```

也可以从输入框工具条的工作流按钮切换——点开是一个下拉菜单,可直接选择 关闭 / 开启 / Ultracode(当前项带勾)。开启后正常提需求即可(比如「调研这三个方案给我推荐」「做一次完整的代码迁移」),**AI 会自己判断是否值得创建工作流**,简单任务仍然内联完成。

**你能感知到的能力**:阶段化执行、并行探索、多 Agent 协作、内置代码审查 / 验证 / 校验和有界的自动修复循环;崩溃或重启后会保守恢复(不会静默完成、不重复已完成步骤、不自动放行审批);可以选运行位置(当前目录 / 新隔离工作树 / 已有工作树);完成的工作流可以「保存为模板」复用。

> 开启工作流模式**不等于立即运行**,只是给了 AI「下一轮可以自主编排」的能力。无痕会话不能用工作流。工作流的执行仍然走正常的[权限审批](07-工具与权限.md)、沙箱和无痕规则。

## 8.3 持续推进(Loop)

按时间、条件或事件**重复触发**同一个任务,每一轮可以继续当前对话,也可以触发绑定目标的工作流。

**怎么用**:

```
/loop 5m 检查部署状态                    # 每 5 分钟跑一次
/loop 检查部署状态                        # 让 AI 自己决定每轮间隔(自定节奏)
/loop until CI is green every 5m: 修复失败的测试   # 条件触发
/loop status          # 查看所有持续推进
/loop pause|resume|stop <id>
```

也可以从工作台的「持续推进」中心用模板创建(检查 CI、刷新报告、进展总结等),选触发方式,选「继续对话」还是「按工作流执行」。

**预算与保护**(创建时可选):最多触发次数、最长运行窗口、token 预算;还有**无进展保护**——每轮结束用确定性判断是否真有进展(不把「跑了一次」当进展),连续无进展会先退避、达到上限就阻塞等你确认。

> 无痕会话不能创建持续推进任务。持续推进不会给 AI 额外的权限捷径——每一轮仍然走会话原有的权限模式和沙箱。自定节奏的持续推进,AI 能选的间隔被限制在 1 分钟到 1 小时之间。
>
> **持续推进 vs 自我唤醒**:持续推进是重复触发;如果 AI 只想「过一会儿把自己叫回来续一次」,用的是一次性的[自我唤醒](09-多Agent与定时任务.md#95-自我唤醒schedule_wakeup)。

## 8.4 执行模式(Mode)

设定本会话 AI 推进任务时的「观察 / 计划 / 验证 / 修复」强度。它既不管定时,也不管编排,只管强度。

**怎么用**:`/mode`、`/mode off|guarded|deep|autonomous`。

| 模式 | 行为 |
| --- | --- |
| 关闭(off) | 不注入推进策略 |
| 守护(guarded) | 稳步推进;验证失败会记录修复事件,重复失败 / 无进展可阻塞 |
| 深入(deep) | 同守护,但允许更深入的探索与验证 |
| 自主(autonomous) | 最强自治;创建 / 运行必须显式给出运行时和输出预算,否则阻塞 |

## 8.5 计划模式(Plan Mode)

对于不那么简单的任务,可以让 AI **先把方案写清楚、你审批后再动手**。方案是「设计契约」,任务清单是「进度真相」,两者独立。

**进入永远由你拍板**:

- 你直接进:输入框的计划按钮,或 `/plan enter`。
- AI 建议进:它识别到复杂任务时会弹一个 Yes/No 对话框,你接受才进,拒绝就直接做。

**五个状态**:关闭 → 制定计划(只能改方案)→ 审批中(方案锁定)→ 执行中(方案冻结,用任务清单追踪进度)→ 完成。没有「暂停」状态,长时间挂起就退出(`/plan exit` 是任何状态下的逃生口),需要时重进。

**方案文件**是自由的 Markdown 设计文档,审批后冻结、执行期不改;右侧计划面板渲染它,支持选段评论和版本历史。**任务清单**是执行期唯一的进度真相。

> **Git 检查点**:审批进入执行时会自动建一个 Git 检查点,完成或退出时清理,你可以随时回滚到这个点。

**设置(设置 → 计划模式)**:

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 计划子 Agent 模式 | 关 | 用一个独立子 Agent 来制定计划 |
| 提问等待超时 | 关(永不超时) | 计划期提问是否超时,以及超时秒数 |

---

## 8.6 任务(Task)

任务是上面这些能力在执行时产生的**用户可见进度**,三种状态:待办 / 进行中 / 已完成。AI 会自动创建和更新任务清单,显示在输入框上方;你也可以手动点完成。

系统还会把各个控制面汇总成一个会话级的活动状态——区分「等你处理」(需要审批 / 选择 / 确认完成)和「等外部」(等某个子任务 / 文件 / 定时器),只有前者才会提示你需要介入。
