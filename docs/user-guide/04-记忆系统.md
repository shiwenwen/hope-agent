# 04 · 记忆系统

记忆系统让 AI **跨会话记住你**——你的偏好、项目背景、长期习惯。你只需要理解三件事:**始终记住**(核心记忆)、**相关时想起**(按需召回)、**从对话中学习**(自动 / 手动沉淀)。更进阶的画像、结构化整理都收在高级区,默认不用管。

本章末尾还介绍与记忆相关的[行为感知](#46-行为感知)。

**总入口**:设置 → **记忆**(含 概览 / 记忆设置 / 记忆管理 / 梦境(Dreaming) / 画像 / 记忆断言(Claims) 六个标签)。

## 4.1 三层记忆:全局 / Agent / 项目

同一条记忆可以属于三个层级之一,优先级 **项目 > Agent > 全局**:

- **全局**——所有会话都能用到(比如「我叫 X、习惯用中文回复」)。
- **Agent**——只有用这个 Agent 的会话能用到(不同助手互不串味)。
- **项目**——只在该项目下的会话可见,**刻意不会泄漏到无关会话**(比如某项目的技术决策)。

**入口**:设置 → 记忆 → 记忆管理,可按作用域筛选、增删改。对话里也可以用 `/remember` 保存、`/forget` 删除、`/memories` 查看。

---

## 4.2 核心记忆与自动记忆索引

每个层级(全局 / Agent / 项目)各有一份 `MEMORY.md` **索引**加若干**主题正文**:

- **核心记忆(MEMORY.md 索引)** 是一份短小稳定的清单(上限约 200 行 / 25 KB),它会始终进入 AI 的上下文,保证「始终记住」的内容可靠、可人工编辑。
- **主题正文**存放详细内容,由 AI 按需读取,不会一直占用上下文。

你可以在设置 → 记忆 → 记忆管理里直接查看和编辑三层的核心记忆。

> AI 的自动学习流程**只能「建议」把内容提升到核心记忆,不能悄悄改写你维护的核心记忆**——这是一条安全底线。关闭自动学习也不会删掉已有的记忆文件。

**相关设置**:

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 记忆总开关 | 开 | 关闭后,AI 侧的核心记忆 / 召回 / 学习 / 记忆工具全部归零(仅保留你自己的管理界面) |
| 核心记忆开关 | 开 | 停止注入核心记忆,但不删文件 |
| 核心记忆预算 | 1600 token | 稳定注入的上限(可选 1000 / 1600 / 2400 或自定义) |

## 4.3 记忆召回:让 AI 想起相关内容

「召回」指在对话中把相关的旧记忆找出来注入。有三种形态:

- **模型按需召回(始终可用)**——即使自动召回全部关闭,AI 仍然可以自己调用记忆工具来查,返回**完整原文**。
- **快速召回(Fast Recall,需你手动开启)**——开启后每一轮都做一次零成本(不调用模型)的检索,自动挑几条最相关的记忆注入。**默认关闭**。
- **深度召回(Deep Recall,次级开关)**——在快速召回结果上再用模型重排提炼,更准但增加延迟和成本。**默认关闭**,且只有开了快速召回才能用。

**入口**:设置 → 记忆 → 概览的「召回」卡片。

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 自动召回相关记忆 | **关** | 是否每轮自动做快速召回 |
| 纳入结构化 claim | 开(召回开启时才生效) | 快速召回是否包含结构化整理过的事实 |
| 深度召回 | **关** | 是否用模型重排(更准、更慢) |

> 关闭自动召回**不等于**关闭记忆——AI 仍然会用核心记忆,也仍然能主动查询记忆工具。

## 4.4 从对话中学习

AI 可以在对话中自动提取值得记住的事实。有三种模式(设置 → 记忆 → 概览的「学习」卡片):

- **自动学习(默认)**——自动提取,作用域清晰的普通条目直接入库,冲突 / 敏感 / 拿不准作用域的进入待审队列。
- **先审核**——所有自动记忆都先进待审队列,你批准前对 AI 不可见。
- **仅手动**——不自动提取,只有你显式保存才写入。

自动提取在后台进行、不打断聊天(累计一定对话量后触发)。待审队列在「管理」标签里,显示待确认的数量。

## 4.5 进阶:离线整理、画像与纠错(Dreaming)

Dreaming 是**离线记忆整理**能力,在应用空闲、定时或手动触发时,把零散记忆整理成可审计、可纠正的长期心智。它默认开启(定时触发默认关),对普通用户是「后台自动变聪明」,你也可以完全不管它。

- **梦境日记(Dream Diary)**——每次整理写一篇 Markdown 叙事,记录本轮「记住 / 固化」了什么,可在 Dashboard 的 Dreaming 标签翻看。
- **结构化 Claim**——把零散句子升级为带证据、作用域、时效、置信度的结构化事实;过期会自动抑制,有冲突会进入待审而不是自动覆盖。在设置 → 记忆 → Claims 里查看管理。
- **用户画像(Profile)**——自动合成一份可读的画像摘要(你的沟通风格、工作习惯、长期偏好),在设置 → 记忆 → 画像里查看。
- **纠错闭环(Lucid Review)**——**记忆只有你能改,AI 不能自行改动**。你可以:
  - **编辑**一条 claim(改内容 / 批准 / 标为过时 / 换作用域 / 置顶);
  - **遗忘**一条 claim(归档保留审计,或彻底删除)。
  - 你的每次纠正都是「最高权威」。

**设置**(设置 → 记忆 → Dreaming,中风险):

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| Dreaming 总开关 | 开 | 是否启用离线整理 |
| 空闲触发 | 开 / 30 分钟 | 空闲达到阈值自动运行 |
| 定时触发 | **关** / 每天凌晨 3 点 | 定时夜间整理 |
| 用户画像合成 | 开 | 是否生成画像摘要 |

> Dreaming 负责「生成 / 整理」记忆,但对话中是否**用到**这些整理结果,仍由 [4.3 的召回开关](#43-记忆召回让-ai-想起相关内容)决定。

> **语义检索**:记忆的按意思检索依赖一个 Embedding(向量)模型,详见 [02 · 记忆向量模型](02-模型与Provider.md#210-记忆向量模型embedding)。
>
> **无痕会话**:[无痕会话](03-对话与会话.md#36-无痕会话)对记忆系统完全「失明」——不注入、不提取、不入库。

## 4.6 行为感知

行为感知让每个会话的 AI 知道你「同时在其它并行会话里做什么」,从而理解「上次那个 bug」「另一个窗口在调的 CI」这类跨会话指代。

- **默认关闭**(整体总开关默认关)。开启后有两档模式:`结构化`(零成本,从数据库聚合出其它会话的列表)、`LLM 摘要`(额外用模型生成自然语言摘要,有成本)。
- **入口**:设置 → 对话与上下文 → 行为感知;每个会话也可用输入框温度滑块旁的**眼睛图标**单独覆盖;对话里可用 `/awareness` 命令。
- 默认排除定时任务、IM 和子 Agent 会话;无痕会话完全不参与。

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 总开关 | **关** | 是否启用行为感知 |
| 模式 | 结构化 | 关 / 结构化 / LLM 摘要 |
| 最多关联会话数 | 6 | 最多提及几个其它会话 |
| 回看窗口 | 72 小时 | 回看多久内的会话 |
