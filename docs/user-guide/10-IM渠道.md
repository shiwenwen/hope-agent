# 10 · IM 渠道

把 Hope Agent 接入你常用的即时通讯工具,就能在 Telegram、飞书、微信等平台里直接和你的 Agent 对话,拥有和桌面端相同的能力(工具、记忆、流式回复)。图片、语音、文件也能直接进入多模态上下文。

## 10.1 支持哪些渠道

目前支持 12 个渠道:

| 渠道 | 认证方式 | 备注 |
| --- | --- | --- |
| **Telegram** | Bot Token | 媒体支持最完整,打字机式流式 |
| **Discord** | Bot Token | 私聊 / 群 / 论坛 / 频道 |
| **Slack** | Bot Token + App Token | 全类型媒体 |
| **飞书 / Lark** | App ID + App Secret | 支持卡片式流式 |
| **QQ 机器人**(官方) | App ID + Client Secret | 部分场景需公网 URL |
| **微信** | 扫码登录 | 私聊,加密媒体 |
| **WhatsApp** | 桥接服务 URL + Token | 经桥接 |
| **Signal** | 手机号 + 链接设备 | 需 signal-cli |
| **iMessage** | macOS 本地 | **仅 macOS** |
| **LINE** | Channel Token + Secret | 部分场景需公网 URL |
| **Google Chat** | Service Account | 部分场景需公网 URL |
| **IRC** | Nick + NickServ | 纯文本,媒体走下载链接 |

> 不支持原生媒体的渠道会自动降级为「贴一个下载链接」的纯文本兜底,不会丢消息。

## 10.2 添加账号

**入口**:设置 → IM 渠道 → 添加账号 → 选渠道 → 填该渠道的凭据(比如 Telegram 填 Bot Token,可即时验证)。

在账号编辑里可以配置:

- **绑定 Agent**——该账号默认用哪个 Agent 回复。
- **私聊策略**——开放(任何人可私聊)/ 白名单(仅指定用户)。
- **群策略与用户白名单**——控制谁能用。
- **自动审批工具**、**接管通知**、**启动上线提醒**、**语音自动转写**、**知识空间访问**、**回复模式**、**思考显示**等。

## 10.3 多模态、审批与回复模式

### 多模态

支持媒体的渠道都可以收发图片、视频、音频、文件。入站媒体先经权限校验,通过后才真正下载并存到本机。语音消息可以开「[自动转写](02-模型与Provider.md#211-语音转写stt)」(默认关,消耗语音识别配额)。

### 工具审批

- 支持按钮的渠道(Telegram / 飞书 / Discord / Slack / QQ / LINE / Google Chat)走**原生按钮**审批;其余渠道走**文本回复**审批。
- **自动审批工具**(账号级开关,默认关):开启后跳过工具审批。如果被跳过的调用本该命中最严格的审批,会记一条审计告警。

### 回复模式(imReplyMode)

三种模式对所有渠道生效,在账号编辑里设,或用 `/imreply` 命令切换:

| 模式 | 行为 |
| --- | --- |
| **分条(split,默认)** | 每轮的叙述和媒体按时序独立发消息;支持流式的渠道每轮是真的打字机效果 |
| **最终(final)** | 只发最后一轮的文本,末尾发媒体,不启用流式 |
| **预览(preview)** | 支持流式的渠道渲染一条不断增长的合并消息;不支持的自动降级为「最终」 |

默认丢弃 AI 的思考过程,`/reason on` 可以打开。

## 10.4 会话接管与交接

每个 IM 聊天(渠道 + 账号 + 聊天 + 话题)在任意时刻只关联**一个**会话,反过来每个会话也只能被一个 IM 聊天接管(双向 1:1)。

- 新的聊天通过 `/session <id>` 或桌面端交接接管时,旧的关联会被解除,并给被踢的聊天发一条「会话已被另一端接管」的通知(可静音)。接管后会立刻补上一轮已完成的回复。
- **实时流式镜像**:你在桌面 / 网页触发的对话,如果这个会话关联了某个 IM 聊天,会实时镜像到 IM 端(按回复模式呈现)。两个通道相互独立——桌面永远走自己的流,不受 IM 回复模式影响。
- 后台任务、子 Agent、定时任务完成的结果也会回投到关联的 IM 聊天或定时任务的投递目标。

## 10.5 IM 里的斜杠命令

IM 里也能用[斜杠命令](03-对话与会话.md#39-斜杠命令大全),但有些行为不同:

| 命令 | 在 IM 里的作用 |
| --- | --- |
| `/sessions` | 打开会话选择器 |
| `/session [<id>\|exit]` | 无参看会话信息;`<id>` 接管;`exit` 解绑 |
| `/project <名>` | 把当前会话归入项目 |
| `/kb [on\|off]` | 群聊里确认知识空间访问;私聊只报状态 |
| `/imreply` / `/reason` | 回复模式 / 思考显示 |
| `/status` | 会话状态,含关联的 IM 渠道和 Agent 来源 |
| 其它 | `/help` `/new` `/clear` `/model` `/search` `/recap` 等大多可用 |

> **IM 里禁用** `/agent`(容易导致 Agent 归属漂移,请去设置里改绑定)和 `/handover`(在 IM 端没有意义)。

## 10.6 知识空间访问

IM 渠道**默认零知识空间访问**。要放开需要两层确认:

1. 在桌面设置里对该账号开启「知识空间访问」(仅所有者可设,默认关)——用于私聊;
2. 群聊里还需要在群内用 `/kb on` 逐个确认。

即便开启,访问仍然受挂载、无痕、外部只读等规则约束。详见 [05 · 知识空间 · 访问控制](05-知识空间.md#58-访问控制)。
