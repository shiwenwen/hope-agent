## 3.1 聊天界面

- **发送**:`Enter` 发送,`Shift+Enter` 换行。
- **停止**:生成过程中,发送按钮会变成方形「停止」按钮,点它可以中断当前回复(等同于 `/stop`)。
- **翻历史**:光标在空输入框时,按 `↑` / `↓` 可以浏览你之前发过的消息(类似终端历史)。
- **忙时排队**:AI 正在回复时你又发消息,会进入「待发送队列」——可以编辑、删除,或作为下一个独立回合发出。是否在空闲时自动发出由设置项「自动发送排队消息」控制(默认开)。
- **上下文用量条**:输入框底部有一条细进度条(绿 → 黄 → 红),显示当前对话占用了多少上下文窗口,鼠标悬停显示「已用 / 总量」。
- **快捷切换**:`Shift+Tab` 可以快速循环切换[权限模式](07-工具与权限.md)(默认 → 智能 → 全自动)。

## 3.2 发送附件与多模态

Hope Agent 支持图片、文件、语音三类输入,随消息一起交给模型理解。

- **图片 / 文件**:点输入框工具栏的「+」按钮选择,或**直接粘贴 / 拖拽**。
- **粘贴长文本**:粘贴一大段文字会自动转成一个文本附件,避免撑爆输入框。
- **语音**:点麦克风按钮录音转文字;也可以在应用窗口内**按住 `Ctrl+Shift+H` 说话**,松开自动转写并插入到光标处。录音时显示波形和实时耗时;支持流式的转写服务会边说边出字。

> 语音功能需要先在[设置里配置语音转写(STT)](02-模型与Provider.md#211-语音转写stt),未配置时会提示。单个文件默认上限 20 MB,单条消息最多 64 个文件。

## 3.3 @ 提及与快捷输入

在输入框里用几个特殊符号可以快速引用东西:

| 输入 | 作用 |
| --- | --- |
| `/` | 触发[斜杠命令](#39-斜杠命令大全) |
| `@` | 提及一个**技能**(插入玫瑰粉色技能标签,发送时把技能说明注入本轮)或**当前工作目录的文件** |
| `[[ ]]` | 提及一篇[知识空间笔记](05-知识空间.md),发送时注入笔记内容 |
| `#` | 插入你预置的**快捷提示**(可复用的提示词片段) |

> - `@skill` 技能提及只对少数固定内置技能开放(办公三件套、数据分析、浏览器控制、macOS 控制),且主对话里需要先在设置里开启「技能提及」(默认关)。
> - `[[note]]` 笔记提及和 `@` 文件提及注入的内容,都被当作「不可信外部数据」处理,不会被当成对 AI 的指令。

## 3.4 思考块、工具块与右侧面板

### 思考块(Thinking)

模型的思考过程会显示成一个可折叠的「思考块」,带大脑图标和**实时耗时**(流式时每 100 毫秒刷新,结束后显示最终耗时)。可以点击折叠 / 展开;是否默认展开由设置项「自动展开 Think 过程」控制。

### 工具调用块

AI 每次调用工具(读文件、执行命令、搜索等)都会显示一个块,含工具名、参数、结果、**耗时**和成功 / 失败状态。连续多个已完成的工具步骤会自动折叠成一个「已处理」分组,保持界面整洁。

一些特殊工具有专属卡片,比如提问卡(`ask_user_question`)、计划卡、任务清单、技能激活块、子 Agent 块等。

### 右侧面板

聊天界面右侧一次只显示一个上下文面板(互相排斥),可以拖拽边缘调宽、可最大化:

- **文件 Diff 面板**——展示 AI 对文件的改动(分屏 / 合并视图),详见 [07 · 工具与权限](07-工具与权限.md)。
- **工作台(Workspace)**——聚合当前任务进度、读写过的文件、访问过的网址、Git 状态、目标等,点输入框上方的状态条进入。
- **计划面板(Plan)**——[计划模式](08-自主任务.md#85-计划模式plan-mode)下的设计文档。
- **浏览器 / macOS 控制 / Canvas / 团队 / 后台任务 / 文件浏览器 / 文件预览 / Pull Request**——对应各自的功能,详见相关章节。

### 消息渲染

回复支持完整 Markdown、代码高亮、数学公式(KaTeX)、Mermaid 图表。在桌面版里,AI 还能写可点击的本地路径链接(点击在文件管理器中打开)。

### 相关设置(设置 → 对话与上下文 → 基础)

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 自动发送排队消息 | 开 | AI 忙时你发的消息是否自动排队发送 |
| 自动展开 Think 过程 | 开 | 思考块流式时是否默认展开 |
| 完成后折叠中间消息 | 开 | 已完成的回合是否自动折叠 |
| 进度更新 | 开 | AI 是否在关键时刻给出简短进度更新,并在回合结束时小结 |
| 会话标题 | 开 | 是否用模型自动为会话起标题 |

## 3.5 会话管理与搜索

- **新建对话**:侧边栏「新建」或 `/new`。在项目里新建对话时会先停在草稿态,发出第一条消息才真正保存。
- **会话列表**:侧边栏按更新时间倒序,可置顶。分「会话 / 子 Agent」两个浏览标签。定时任务、无痕、知识空间、设计空间的会话**不出现在主列表**。
- **切换 / 改名 / 删除**:点击切换;可手动改名(手动名不会被自动标题覆盖);删除会一并清理消息、附件等。
- **在新会话中继续(Fork)**:把当前会话复制成一个新的独立会话(复制对话内容和配置,但不复制运行中的目标、持续推进、后台任务)。

### 搜索

- **全局搜索**(侧边栏):全文检索所有会话,命中片段高亮。
- **会话内搜索**(`Cmd+F` / `Ctrl+F`):只搜当前会话,命中后自动滚动定位并高亮。

### 未读怎么算

- **一个会话不管有多少条未读消息,都只算「1 个未读会话」**。会话行显示一个小圆点;项目、全局和 Dock 上显示的是未读会话的**数量**;系统托盘只显示有没有红点。
- 只有当聊天主视图选中该会话、窗口在前台、页面可见、且消息列表停在最新位置时,它才算「已读」。
- 定时任务和 IM 渠道各有独立的未读统计,普通对话的「全部已读」不会清掉它们。

## 3.6 无痕会话

无痕会话是**不留痕迹的临时对话**,关闭即焚。

**怎么开**:点输入框工具栏或标题栏的**幽灵图标**。

无痕会话会做这些事来保护隐私:

- 切走当前无痕会话时立即彻底删除,不留任何记录。
- **不注入记忆、不做行为感知、不自动提取记忆、不进知识空间被动召回**——记忆系统对它完全「失明」。
- 不进入侧边栏列表、全局搜索和 Dashboard 统计。

> **限制**:无痕会话与「项目」「IM 渠道」「工作流模式」「进行中的目标」互斥——已经归入项目或绑定 IM 的会话无法转无痕(开关会灰化并说明原因)。无痕会话里仍然可以设置会话工作目录。

## 3.7 会话工作目录

给当前对话指定一个默认操作目录,它会:

- 注入到 AI 的提示词里,告诉它默认在哪读写文件;
- 作为 `exec`(执行命令)的实际工作目录、`read`(读文件)相对路径的解析根;
- 目录的顶层文件清单也会告诉 AI,方便它感知你在做什么。

**怎么设**:点输入框工具栏的「工作目录」按钮。桌面版弹出系统目录选择器;网页 / 服务器版弹出目录浏览框(指向服务器机器上的路径)。

优先级是「会话设定 > 项目设定 > 默认工作区」。项目里的会话总是有一个工作目录(默认是 `~/.hope-agent/projects/{项目}/workspace/`)。

## 3.8 长对话的上下文压缩

模型能记住的对话长度是有限的(上下文窗口)。对话变长、接近上限时,Hope Agent 会**自动分层压缩**,让长对话和长任务能继续,而不会突然「失忆」或报错。压缩过程是自动的,你通常只会在顶部看到一条压缩提示。

压缩从「零成本清理过时的工具结果」开始,逐级到「摘要旧历史」;摘要之后还会自动把最近编辑过的文件的当前内容重新注入,保证 AI 不会忘记正在改的文件。

**手动命令**:

- `/compact`——手动压缩当前对话。
- `/context`——查看当前上下文占用明细。

**主要设置(设置 → 对话与上下文 → 上下文压缩)**:

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 启用上下文自动压缩 | 开 | 关闭后只保留最基础的截断与兜底 |
| 反应式微压缩 | 开 | 每轮结束时按需清理旧的工具结果 |
| 触发比例 | 0.75 | 上下文用量超过此比例时触发反应式清理 |
| 摘要模型 | 空(用对话模型) | 生成摘要用的模型,可指定更便宜的模型省成本 |
| 摘要触发阈值 | 0.85 | 用量超过此比例时触发摘要压缩 |

> 这些属于「行为调整」类设置(中风险),会影响上下文和成本。绝大多数用户保持默认即可。压缩还有一整套更细的参数(裁剪比率、保留最近轮次、文件恢复等),都在同一面板的「高级」区,按需调整。

## 3.9 斜杠命令大全

在聊天输入框或任意 IM 渠道里输入 `/` 就能触发斜杠命令。支持模糊匹配。下面是全部内置命令(每个可安装的技能还会自动生成一个 `/技能名` 命令)。

### 会话

| 命令 | 用途 |
| --- | --- |
| `/new` | 新建会话 |
| `/clear` | 删除当前会话的所有消息 |
| `/compact` | 压缩当前会话上下文 |
| `/stop` | 停止当前回复 |
| `/rename <标题>` | 重命名会话 |
| `/export` | 把当前会话导出为 Markdown |
| `/status` | 查看会话状态(Agent / 模型 / ID / 消息数) |
| `/usage` | 查看当前会话 Token 用量 |
| `/context` | 查看上下文窗口占用明细 |
| `/sessions` | 打开会话选择器(可带关键词搜索) |
| `/session <id>` | 切换 / 关联会话 |

### 项目与目标

| 命令 | 用途 |
| --- | --- |
| `/project [名称]` | 打开项目选择器,或进入某项目新建会话 |
| `/projects` | 列出所有项目 |
| `/goal <目标> --criteria <完成标准>` | 设定 / 管理会话目标,详见 [08 章](08-自主任务.md#81-目标goal) |
| `/plan` | 进入 / 管理[计划模式](08-自主任务.md#85-计划模式plan-mode) |
| `/workflow [on\|off\|ultracode]` | 开关[工作流模式](08-自主任务.md#82-工作流workflow) |
| `/mode <off\|guarded\|deep\|autonomous>` | 设置[执行模式](08-自主任务.md#84-执行模式mode) |
| `/loop <指令>` | 创建 / 管理[持续推进任务](08-自主任务.md#83-持续推进loop) |

### 模型与思考

| 命令 | 用途 |
| --- | --- |
| `/model [名称]` | 列出或切换模型 |
| `/models` | 列出所有可用模型 |
| `/thinking <off\|low\|medium\|high\|xhigh>` | 设置思考强度(别名 `/think`) |

### 记忆

| 命令 | 用途 |
| --- | --- |
| `/remember <内容>` | 保存一条记忆 |
| `/forget <关键词>` | 搜索并删除最匹配的一条记忆 |
| `/memories` | 列出记忆 |

### Agent 与团队

| 命令 | 用途 |
| --- | --- |
| `/agent <名称>` | 切换 Agent(自动新建会话;IM 里禁用) |
| `/agents` | 列出所有 Agent |
| `/team` | 管理 [Agent 团队](09-多Agent与定时任务.md#92-agent-团队) |

### 权限与工具

| 命令 | 用途 |
| --- | --- |
| `/permission <default\|smart\|yolo>` | 设置[权限模式](07-工具与权限.md#72-权限模式三选一) |
| `/review` | 对未提交的代码改动运行本地审查 |
| `/search <查询>` | 让 AI 帮你搜索 |
| `/recap [--range=7d\|--full]` | 生成[复盘报告](12-项目与数据洞察.md#124-recap-复盘报告) |
| `/awareness <off\|on\|mode ...>` | [行为感知](04-记忆系统.md#46-行为感知)开关 |
| `/prompts` | 查看当前 Agent 的完整系统提示词 |
| `/help` | 显示所有命令 |

### IM 渠道专用

| 命令 | 用途 |
| --- | --- |
| `/imreply <split\|final\|preview>` | 设置 IM 回复模式 |
| `/reason <on\|off>` | IM 输出是否包含思考过程 |
| `/kb <on\|off>` | 群聊里确认[知识空间访问](05-知识空间.md) |
| `/handover <渠道:账号:会话>` | 桌面端把会话推送到某 IM 会话 |

> 更多 IM 相关的命令行为差异见 [10 · IM 渠道](10-IM渠道.md)。
