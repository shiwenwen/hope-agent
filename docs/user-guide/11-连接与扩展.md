## 11.1 MCP:接入外部工具

MCP(Model Context Protocol)是一个开放标准,让 AI 助手能连接各种外部「工具服务器」。Hope Agent 内置了完整的 MCP 客户端,可以把外部服务器提供的工具、资源和提示词直接注入对话。

- **四种连接方式**:stdio、Streamable HTTP、SSE、WebSocket。
- **完整 OAuth 2.1 支持**:需要授权的服务器可以自动走 OAuth 登录,凭据安全落盘。

**怎么添加**(设置 → MCP 服务器):

- 左栏是服务器列表(带状态、连接方式、工具数),右栏是编辑页。可以配名称、启用、信任等级、连接方式、工具白 / 黑名单、超时、自动审批(仅对信任等级为『已信任』的服务器可开启)。
- 点「测试连接」实时看结果和工具数;需要授权的服务器会显示「授权」按钮触发 OAuth。
- 支持从 `claude_desktop_config.json` 一键导入。

> MCP 服务器配置属于**高风险**——它可能带凭据。服务器配置本身(含密钥、环境变量)只能在界面里改,AI 不能通过对话修改。工具默认会全部注入;对于工具很多的大型服务器,可以开启「延迟加载」,改成按需发现,避免占用上下文。

## 11.2 把 Hope Agent 当作 MCP 服务器

反过来,Hope Agent 也可以**作为 MCP 服务器**,让本机上的其它 AI 工具(Claude Code、Cursor 等)接入它的能力。

- **入口**:命令行运行 `hope-agent mcp`。
- 默认**只读**,加 `--allow-writes` 才暴露写工具。
- 出于安全,它**永远不暴露**这些能力:写用户代码仓库、部署、分享、删除、导出。

## 11.3 Hooks:生命周期钩子

Hooks 让你在关键节点(工具调用前后、会话开始 / 结束、上下文压缩、权限决策等)上挂自定义处理器。它字段级对齐 Claude Code 的 hooks 协议,社区脚本可以直接用。

**入口**:设置 → Hooks。选一个事件,加匹配规则,再加处理器。

**五种处理器**:

| 处理器 | 作用 |
| --- | --- |
| command | 跑一个 shell 命令 |
| http | POST 到一个 URL |
| mcp_tool | 调用某个 MCP 工具 |
| prompt | 做一次性的 LLM 调用,结果作为附加上下文 |
| agent | 派生一个子 Agent |

**四层作用域**(全部叠加生效):

- **用户 / 托管**——全局生效。
- **项目 / 本地**——按会话工作目录生效,**默认关闭**(供应链防护),需要在设置里显式开启「允许项目级钩子」。

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 全局禁用所有钩子 | 关 | 一键关掉所有钩子 |
| 允许项目级钩子 | 关 | 是否让仓库里签入的钩子按目录生效 |

> Hooks 是**高风险**功能(能跑任意命令),所以 **AI 只能读、不能改钩子配置**——只能在界面里改,防止 AI 给自己装上命令执行能力。

## 11.4 技能系统

技能是给 AI 装上的**专项方法论 / 操作手册**——一个目录加一个 `SKILL.md` 文件。系统提示词里只放技能的「名称 + 描述」,AI 按需激活,不常驻占用上下文。

### 内置技能

Hope Agent 内置了一批技能,按类别:

- **平台自管理**:`ha-settings`(用对话改设置)、`ha-skill-creator`(创建 / 编辑技能)、`ha-find-skills`(发现安装第三方技能)、`ha-browser`(浏览器自动化方法)、`ha-mac-control`(macOS 控制方法,仅 macOS)、`ha-knowledge`(知识空间工作法)、`ha-logs` / `ha-data-stores`(本地数据只读排查)、`ha-self-diagnosis`(自诊断 + 提交 GitHub issue)、`ha-self-update`(检查安装更新)。
- **编程方法论**:实现、计划、调试、测试策略、代码审查、多 Agent 协作、完成验证、工作流编写等 8 个(会按任务自动推荐组合)。
- **办公三件套**(需 python3):`office-docx`(Word)、`office-xlsx`(Excel)、`office-pptx`(PowerPoint)——创建、编辑、检查、交付,支持真实表格、图表、公式、批注修订等。
- **办公方法论**:会议纪要、邮件起草、周报月报、图表绘制。
- **集成**:`feishu`(飞书云文档 / 多维表格 / 云盘 / 知识库 / 审批 / 日历 / 联系人 / 招聘等 30 多个工具)、`ha-data-analytics`(本地数据分析 + 生成可分享报告)。

### 三种激活方式

| 方式 | 怎么用 | 差异 |
| --- | --- | --- |
| **AI 自主** | AI 调用 `skill` 工具 | 主入口。普通技能把说明注入主对话;标记 `fork` 的技能在独立子 Agent 里跑,只回一段摘要(不污染主对话) |
| **斜杠命令** | 输入 `/技能名 [参数]` | 你主动触发 |
| **`@skill` 提及** | 输入框 `@` 菜单选技能 | 只对少数固定内置技能开放(办公三件套、数据分析、浏览器、macOS 控制),主对话需先开启「技能提及」 |

### 创作与安装

- **创作**:用 `ha-skill-creator` 技能,或在「设置 → 技能」里创建。AI 自主创建的技能会进入**草稿**状态,不直接生效,等你审核(经过安全扫描等多道闸门)。
- **安装第三方技能**:`ha-find-skills` 会从 ClawHub、Skillhub(更适合中国大陆用户)或 GitHub 搜索发现。**安装第三方技能是高风险操作**(第三方代码进入 AI 的工具链),AI 必须先给你看候选并**显式确认**才会安装。
- **兼容标准**:遵循 [agentskills.io](https://agentskills.io) 开放标准,也兼容 Claude Code、OpenAI Codex 等生态的技能,可以直接导入。
