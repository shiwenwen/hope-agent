# Hope Agent 0.8.0 — Desktop Slimming & Speedups, plus Stability Fixes

> [简体中文](https://github.com/shiwenwen/hope-agent/blob/v0.8.0/docs/release-notes/v0.8.0.md) · **English**

> Release date: 2026-06-09

> See [`CHANGELOG.md`](https://github.com/shiwenwen/hope-agent/blob/v0.8.0/CHANGELOG.md#080---2026-06-09) for the full list.

v0.8.0 is a maintenance release focused on performance and stability: the desktop app gets a systematic slimming and startup speedup pass, alongside fixes for several issues that affected real use — macOS system permissions being reset on every update, and the Windows release build breaking.

## Highlights

- **Desktop slimming and startup speedup**: the frontend main bundle shrinks by ~60% — translation files and the Settings view are now lazy-loaded on demand instead of being bundled into the first paint; the Rust release build enables size-optimized compilation (thin LTO + symbol stripping) for a smaller binary; startup-time work such as FTS index rebuild, embedding-model initialization, and weather refresh is deferred to the background, so cold start is faster and the first screen becomes interactive sooner; the session database gains a read-only connection pool, so history search (Cmd+F / session search) is no longer blocked by writes while a reply is being generated.

## Fixes

- **macOS system permissions no longer reset on update**: release bundles are now signed with a stable self-signed certificate, so granted permissions such as screen recording and accessibility are no longer reset by macOS on each auto-update and shown as "not authorized".
- **Windows desktop build restored**: fixes the Windows release bundle failing to package because OpenSSL could not be compiled from source — WeChat media encryption/decryption now uses a pure-Rust implementation and the `openssl` dependency is removed, so Windows no longer compiles OpenSSL; Linux / Docker self-hosting still statically bundles OpenSSL and is unaffected at runtime.
- **Skill settings text and pending-review badge**: long skill descriptions now truncate correctly within the list instead of overflowing to the right; the self-evolution pending-review draft count is shown as a numeric badge on the left-side skills entry and the Settings sidebar "Skills" item, reflecting the current total pending count rather than only newly-unread items.
- **Add/edit model trims input whitespace**: saving or testing a Provider now automatically trims leading/trailing spaces and newlines from fields such as base URL, model ID, API key, and name, avoiding connection failures caused by whitespace pasted in by accident.
- **Model connectivity test verifies an actual reply**: testing a model no longer treats "HTTP 200 is enough" as a pass — it now requires actual reply content; reasoning models truncated by the test's output cap (e.g. the GPT-5 family) are still correctly judged as connected and no longer falsely reported as failed.
- **IM replies quote the original message only once**: fixes streaming channels such as Telegram and Feishu in split mode quoting the original message on every part when a reply is split into multiple messages — now only the first message of the whole turn carries the quote (consistent across DMs and groups).
