## 亮点

### 统一权限引擎（`permission::engine`）

把以前散落在 8 套机制（`ToolPermissionMode` / `dangerous_skip_all_approvals` / `internal` flag / `auto_approve_tools` / `require_approval` / `exec` allowlist / Plan Mode allowlist / `approval_timeout`）里的判定逻辑收敛到**单一规则引擎 + 不同 preset**，参考 Claude Code 的 "unified rule engine + presets" 思路。

所有工具调用走 `engine::resolve_async(ctx) -> Decision { Allow | Ask { reason } | Deny { reason } }` 单入口判定，优先级从高到低：

```
Plan Mode > Internal > YOLO > Protected/Dangerous > AllowAlways > Session 模式 preset > 兜底 Allow
```

- **Plan Mode 是正交工作模式**——能压过 Global YOLO，是最终拦截层
- **保护路径 + 危险命令**在非 YOLO 下强制弹（`forbids_allow_always=true`，AllowAlways 按钮置灰）
- **YOLO 模式只 `app_warn!` 不弹**——用户既然开了 YOLO 就接受全部风险

### Session 模式三选一

每个会话独立携带 `permission_mode`，标题栏 `PermissionModeSwitcher` dropdown 切换：

| 模式 | 行为 | 谁该用 |
|------|------|--------|
| **Default** | 硬编码"编辑类必审批" + Agent `custom_approval_tools` 叠加 | 大多数用户 |
| **Smart** | `_confidence: "high"` 自报 + LLM judge_model + `Both` 并联 | 信任 LLM 判断的进阶用户 |
| **Yolo** | 该会话内全放行（仅 Plan 仍能拦） | 一次性脚本会话 |

每个 Agent 可在配置里设置「该 Agent 新会话的默认权限模式」（`AgentConfig.capabilities.default_session_permission_mode`）。

### Smart 模式：self_confidence + judge_model

模型可在 tool_call args 里**主动**加 `_confidence: "high"` 表示"高度确信此次安全"，命中直接 Allow（除非命中保护路径 / 危险命令的 strict 层）。Smart 模式开启时 system prompt 末尾自动注入引导文案，告诉模型这个机制怎么用。

`judge_model` 路径：起独立 LLM 判官 side_query，5s 硬超时 + 60s TTL × 256 cap LRU 缓存。Judge 不可达时按 `SmartFallback` 降级（`Default` 保留 Ask / `Allow` 静默放行 / `Ask` 显式弹）。

### 保护路径 / 危险命令 / 编辑命令三个独立列表

文件落 `~/.hope-agent/permission/{protected-paths,dangerous-commands,edit-commands}.json`，缺失则用各模块硬编码的 `DEFAULT_*` 默认值：

- **保护路径**默认包含 `~/.ssh/`、`.env`、`*secret*`、`*.pem` 等 16 项
- **危险命令**默认包含 `rm -rf /`、`git push --force`、`DROP TABLE` 等 23 项
- **编辑命令**默认包含 `rm `、`mv `、`git commit`、`npm install` 等 50+ 项（exec 编辑性子命令）

「设置 → 权限」一级 tab 提供完整 CRUD UI + 「恢复默认」按钮。

### 审批弹窗升级

- **圆环倒计时**：右上角进度环显示剩余时间，最后 30 秒变红
- **Reason banner**：按 6 种 `AskReason`（EditTool / EditCommand / DangerousCommand / ProtectedPath / AgentCustomList / SmartJudge）渲染 i18n 文案
- **Strict 模式 UI**：保护路径 / 危险命令命中时顶部红色 ShieldAlert，AllowAlways 按钮置灰

### 设置面板

- **「设置 → 权限」一级 tab** 容纳：Global YOLO 开关 / Smart 模式配置 / 保护路径 list / 编辑命令 list / 危险命令 list / 审批超时
- **Agent 配置 → 「审批」tab**：「自定义工具审批」开关 + 17 个工具 9 分组勾选清单（仅 Default 模式生效）+ 该 Agent 默认会话权限模式 selector

## 破坏性变更（无数据迁移）

clean break，老用户审批规则需重新设置：

- 删除 `ToolPermissionMode::Auto/AskEveryTime/FullApprove` 枚举 + DB 字段
- 删除 `AppConfig.dangerous_skip_all_approvals` → 迁移到 `AppConfig.permission.global_yolo`
- 删除 `AppConfig.approval_timeout_secs` / `_action` → 迁移到 `AppConfig.permission.{approval_timeout_secs, approval_timeout_action}`
- 删除 `CapabilitiesConfig.require_approval` → 替换为 `enable_custom_tool_approval` + `custom_approval_tools`
- 删除 `ChannelAccountConfig.auto_approve_tools` → 改为消费 channel 绑定 session 的 `permission_mode='yolo'`
- 删除老 `exec-approvals.json` 文件（路径不复用）
- 删除 Tauri 命令 `set_tool_permission_mode` → 改名 `set_permission_mode`
- 新增 12 个 Tauri 命令 + HTTP 路由（permission 域）

完整命令对照见 [`docs/architecture/api-reference.md`](../architecture/api-reference.md) 的 Permission 段。

## 升级建议

1. **首次启动 0.2.0** 后，审批默认走 `default` 模式 + `permission.global_yolo: false`，跟以前的 `Auto` 模式行为接近——大多数用户无感
2. **以前依赖 `Dangerous Mode` 的用户**：现在叫 Global YOLO，CLI flag `--dangerously-skip-all-approvals` 不变；GUI 入口在「设置 → 权限 → Global YOLO」
3. **以前用过 `exec-approvals.json` allowlist 的用户**：老文件不再读取；新版按命令前缀 always-allow 仍然在 exec 内部生效，只是不导入历史条目
4. **想试 Smart 模式的用户**：「设置 → 权限 → Smart 模式」配置 strategy + judge_model（可选）+ fallback；标题栏切到 Smart 看效果
