# Hope Agent 0.22.0 — 内嵌终端、Git/PR 面板与 LSP 诊断

> **简体中文** · [English](https://github.com/shiwenwen/hope-agent/blob/v0.22.0/docs/release-notes/v0.22.0.en.md)

> 发布日期：2026-07-21

> 详见 [`CHANGELOG.md`](https://github.com/shiwenwen/hope-agent/blob/v0.22.0/CHANGELOG.md#0220---2026-07-21)。

v0.22.0 把「在应用里把编码这件事从头做到尾」补得更完整：对话底部内置了一个真正的交互式终端，随手就能跑命令；Git / PR 面板打通了从本地分支到推送、建 PR 的完整闭环；配置了 language server 后，AI 每轮会自动看到工作区的语义诊断，更快发现类型错误。

## 亮点

- **对话底部新增内嵌终端**：一个真正的交互式终端直接内嵌在对话下方，支持多标签、拖拽调整高度、最大化，以及 `⌘/Ctrl+J` 一键开关；新终端会自动进入当前会话或项目 Worktree 的有效工作目录，配色随应用亮色 / 暗色主题切换以保证高对比度，输出按有界窗口重放。桌面与 HTTP / Web 共用同一套核心实现——远程终端默认关闭，受 `filesystem.allowRemoteWrites` 实时权限门保护，一旦撤权会立即终止远程创建的 Shell。
- **Git / PR 面板打通完整交互闭环**：无论当前是 detached HEAD、只在本地还没推送的分支、已推送但还没建 PR、已经有 PR，还是 GitHub 暂时不可用，面板都会给出明确且可操作的界面，而不再让你对着一个空面板猜下一步。只在本地的分支可以一键「推送」或「推送并创建 PR」，创建成功后直接在应用内打开 PR 面板；PR 详情同步延迟或失败时也保留了重试和跳转 GitHub 的回退入口。
- **AI 现在能看到 LSP 语义诊断**：配置了 language server（rust-analyzer / tsserver / pyright 等）后，AI 每一轮都会自动收到当前工作区的语义诊断，本轮刚改过的文件优先呈现，帮它更快发现类型错误与告警，少走一个来回。

## 变更

- **apt / dnf 软件源迁移到新地址 `https://repo.hopeagent.ai`**：安装包体积在 v0.21.0 全线突破了 GitHub 单文件 100MB 的硬上限，旧的 GitHub Pages 软件源结构性停更。apt / dnf 源改由 Cloudflare R2 托管——无单文件上限、下行流量全免，是高频下载软件源更自然的落点。**已经在用旧源（`shiwenwen.github.io/hope-agent-linux-repo`）的用户，需要按 README 重新配置一次源地址**才能继续收到更新；直接下载 `.deb` / `.rpm` / AppImage，或使用桌面版自升级的用户不受影响。

## 修复与稳定性

- **工作台「文件」分区不再把工具产出的文件误标为「读」**：先被读取、之后又被工具（图片生成 / 附件 / exec）产出的文件，在长会话里滚出当前消息窗口后，会正确显示为「输出」并按产出时间排到最前，与窗口内的实时显示保持一致。
- **Git / PR 面板不再无限转圈**：修复了空请求 key 命中 `null === null` 导致的持续加载，网络请求现在只在具备读取条件时才触发，PR 发现与已有 PR 的定时刷新也拆开了各自的节奏。
