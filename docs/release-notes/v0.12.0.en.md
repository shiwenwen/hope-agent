# Hope Agent 0.12.0 — Chrome Extension Browser Backend

> [简体中文](https://github.com/shiwenwen/hope-agent/blob/v0.12.0/docs/release-notes/v0.12.0.md) · **English**

> Released: 2026-06-23

> See [`CHANGELOG.md`](https://github.com/shiwenwen/hope-agent/blob/v0.12.0/CHANGELOG.md#0120---2026-06-23) for the full list.

The headline of v0.12.0 is a new optional **Chrome extension browser backend**: an MV3 extension plus a native-messaging bridge that drives **your already-logged-in, real Chrome** (with all its cookies and sessions) to operate the web — ideal for tasks that need a signed-in state. When the extension is unavailable and the action doesn't depend on real-Chrome state, it falls back automatically to the existing isolated CDP backend. The desktop installer already ships the extension files, so you can install it locally with one click before the Web Store listing goes live. This release also adds searchable look-back over past sessions after compaction, plus fixes to compaction feedback and settings number inputs.

## Highlights

- **Chrome extension browser backend**: optionally drive your already-logged-in, real Chrome (with all cookies and sessions) via an MV3 extension plus a native-messaging bridge — ideal for tasks that need a signed-in state. All eight browser actions (navigate / snapshot / act / observe / control / tabs / profile / status) run through the extension against real tabs, with real pages taken over via an explicit claim. The raw-CDP escape hatch is on by default but **requires approval on every call and can never be "always allowed"**, and dangerous methods (such as reading cookies) are blocked. When the extension is unavailable and the action doesn't depend on real-Chrome state, it falls back automatically to the existing isolated CDP backend. You can turn off raw CDP or switch the backend preference (extension-first / isolated-CDP-only / extension-only) under Settings → Browser → Advanced.
- **Desktop ships the extension for one-click local install**: ahead of the store listing, the desktop installer already bundles the extension's runtime files. From Settings → Browser you can open the extension directory with one click and load it in Chrome via "Load unpacked," with no need to download source yourself. The extension lives in a stable working directory at `~/.hope-agent/extension/browser/`, so app updates or moves don't break an already-loaded extension; the native-messaging host self-registers on desktop startup and authenticates with a broker token, and together with the extension's backoff reconnect it recovers automatically after an app restart — no manual reload required. When the extension is the default but unavailable and the app silently falls back to CDP, a dismissible guidance banner appears at the top of the chat, with a confirmation once the connection succeeds.
- **Searchable look-back over past sessions after compaction**: a new built-in model tool, `sessions_search`, lets the model search persisted messages of the current or past regular sessions by keyword after context compaction, returning a context window around each hit — so compacted context behaves like "soft memory," and exact historical conversation details can still be recovered when needed, without hand-written SQL. Global search covers only regular, non-incognito sessions by default, keeping cron / subagent / IM channel / knowledge-sidebar sessions out. A new bundled `ha-data-stores` skill also documents Hope Agent's local data root, its main SQLite / file stores, and a read-only query workflow, so diagnostics prefer the product's own tools first and query the underlying data safely only when necessary.

## Other improvements

- **Cleaner task lists and context-usage bar**: task lists in messages and Workspace task progress now collapse by default once all tasks are done (still expandable for detail), and the token-usage bar at the bottom of the chat input is now an inset capsule that no longer overflows the rounded corner.
- **More complete manual context-compaction feedback**: compaction triggered from `/compact`, the title bar, the Workspace, and the `/context` card now shares one set of progress / completion / failure system messages, and retains the final result and post-compaction token usage after a refresh or reopening the session; when summary generation times out, it clearly distinguishes "cleaned up but summary timed out" from "summary timed out, not compacted."
- **Fixed number-input editing in Settings**: number inputs in Settings now commit on blur, so a value isn't clamped to its range mid-edit and you can delete the old digits; nullable fields such as weather coordinates can again be cleared back to unset by emptying the input.
