# Hope Agent 0.2.0 — Permission/Approval System Rewrite + Tool Plumbing Cleanup

> Release date: TBD ([Unreleased])

> 0.2.0 is the first cumulative release after 0.1.0. This note focuses on the largest internal + user-facing change: the **permission/approval system v2 rewrite**. For the full set of cumulative [Unreleased] entries (Self-Evolution tab, Diff panel, project-as-first-class sidebar, local model job cards, `/project` slash command, project-bound IM channel, vision tool protocol, etc.), see [CHANGELOG.md](../../CHANGELOG.md#unreleased).

## Highlights

### Unified Permission Engine (`permission::engine`)

We collapsed eight scattered mechanisms (`ToolPermissionMode` / `dangerous_skip_all_approvals` / `internal` flag / `auto_approve_tools` / `require_approval` / `exec` allowlist / Plan Mode allowlist / `approval_timeout`) into a **single rule engine + presets**, inspired by Claude Code's "unified rule engine + presets" model.

Every tool call now flows through `engine::resolve_async(ctx) -> Decision { Allow | Ask { reason } | Deny { reason } }`. Priority order, high to low:

```
Plan Mode > Internal > YOLO > Protected/Dangerous > AllowAlways > Session preset > Default Allow
```

- **Plan Mode is the orthogonal work mode** — it overrides Global YOLO and is the final blocking layer
- **Protected paths + dangerous commands** force a prompt in non-YOLO modes (`forbids_allow_always=true`, AllowAlways button greyed out)
- **YOLO mode only emits `app_warn!`** — opting into YOLO means accepting all the risk

### Session mode picker

Each session carries its own `permission_mode`, switched from the chat title bar dropdown:

| Mode | Behavior | Best for |
|------|----------|----------|
| **Default** | Hard-coded "edit-class must approve" + agent's `custom_approval_tools` overlay | Most users |
| **Smart** | `_confidence: "high"` self-tag + LLM `judge_model` + `Both` strategy | Users who trust the LLM's judgment |
| **Yolo** | Allow everything in this session (only Plan Mode still blocks) | One-shot scripting sessions |

Each Agent can set "default permission mode for new sessions of this Agent" (`AgentConfig.capabilities.default_session_permission_mode`).

### Smart mode: self_confidence + judge_model

The model can proactively add `_confidence: "high"` to tool_call args, signaling "I'm highly confident this is safe." A high-confidence tag short-circuits to Allow (unless the call hits the strict layer — protected path / dangerous command). When Smart mode is on, the system prompt automatically appends guidance telling the model how to use this mechanism.

The `judge_model` path runs an independent LLM judge side_query with a 5 s hard timeout and a 60 s TTL × 256-cap LRU cache. When the judge is unreachable, behavior follows `SmartFallback`: `Default` keeps the Ask, `Allow` silently allows, `Ask` keeps the explicit prompt.

### Three independent pattern lists

Stored under `~/.hope-agent/permission/{protected-paths,dangerous-commands,edit-commands}.json`. Missing files fall back to per-module `DEFAULT_*` constants:

- **Protected paths** default to 16 entries including `~/.ssh/`, `.env`, `*secret*`, `*.pem`
- **Dangerous commands** default to 23 entries including `rm -rf /`, `git push --force`, `DROP TABLE`
- **Edit commands** default to 50+ entries including `rm `, `mv `, `git commit`, `npm install` (exec subcommands considered editing)

The new "Settings → Permission" top-level tab provides full CRUD UI + a "Restore defaults" button per list.

### Approval dialog upgrades

- **Countdown ring** in the top-right shows remaining time, turning red in the last 30 seconds
- **Reason banner** renders i18n copy for all six `AskReason` variants (EditTool / EditCommand / DangerousCommand / ProtectedPath / AgentCustomList / SmartJudge)
- **Strict mode UI**: protected-path / dangerous-command hits show a red ShieldAlert at the top and disable the AllowAlways button

### Settings panels

- **"Settings → Permission" top-level tab** contains: Global YOLO toggle / Smart mode config / Protected paths list / Edit commands list / Dangerous commands list / Approval timeout
- **Agent config → "Approval" tab**: "Custom tool approval" toggle + 17-tool 9-group checklist (Default mode only) + per-Agent default session permission mode selector

## Breaking changes (no data migration)

Clean break — existing users will need to reconfigure their approval rules:

- Removed `ToolPermissionMode::Auto/AskEveryTime/FullApprove` enum + DB column
- Removed `AppConfig.dangerous_skip_all_approvals` → moved to `AppConfig.permission.global_yolo`
- Removed `AppConfig.approval_timeout_secs` / `_action` → moved to `AppConfig.permission.{approval_timeout_secs, approval_timeout_action}`
- Removed `CapabilitiesConfig.require_approval` → replaced by `enable_custom_tool_approval` + `custom_approval_tools`
- Removed `ChannelAccountConfig.auto_approve_tools` → use the bound session's `permission_mode='yolo'`
- Removed legacy `exec-approvals.json` (path not reused)
- Renamed Tauri command `set_tool_permission_mode` → `set_permission_mode`
- Added 12 new Tauri commands + HTTP routes (permission domain)

Full command/route table: see [`docs/architecture/api-reference.md`](../architecture/api-reference.md) Permission section.

## Upgrade notes

1. **First launch on 0.2.0**: approval defaults to `default` mode + `permission.global_yolo: false`, behaving close to the old `Auto` mode — most users won't notice
2. **If you used `Dangerous Mode`**: it's now called Global YOLO. The CLI flag `--dangerously-skip-all-approvals` is unchanged; the GUI entry moved to "Settings → Permission → Global YOLO"
3. **If you used `exec-approvals.json` allowlist**: the old file is no longer read; per-command-prefix always-allow still works inside exec, but historical entries are not imported
4. **Want to try Smart mode**: configure strategy + judge_model (optional) + fallback under "Settings → Permission → Smart mode", then switch the title bar to Smart
