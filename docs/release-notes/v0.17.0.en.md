# Hope Agent 0.17.0 — Vision Bridge and Unified Background Model Fallback

> [简体中文](https://github.com/shiwenwen/hope-agent/blob/v0.17.0/docs/release-notes/v0.17.0.md) · **English**

> Released: 2026-07-10

> See [`CHANGELOG.md`](https://github.com/shiwenwen/hope-agent/blob/v0.17.0/CHANGELOG.md#0170---2026-07-10) for the full list.

v0.17.0 centers on the **Vision Bridge**: when your chat model can't see images, Hope Agent no longer just drops them — a separately configured vision model describes the image as text and injects it into the conversation instead. Around that headline, this release also brings ~15 background model calls (Recap, Dreaming, Knowledge Space compilation, and more) onto a unified cross-model fallback and usage-accounting path, adds automatic OCR archiving for scanned PDFs and real progress feedback for Knowledge Space binding/import, and fixes a synchronous-IO issue that could stall the backend.

## Highlights

- **Vision Bridge**: when your main chat model doesn't support vision (e.g. text-only models like DeepSeek) but receives an image, you can configure a separate vision model to describe the image as text and inject it into the conversation — replacing the previous "drop the image, leave a placeholder" fallback. Off by default; enabled only once a vision model is explicitly configured. Covers both user-uploaded images and tool-produced images (browser screenshots, image-generation results, PDF pages). Usage is tracked separately under a "vision description" category in the Dashboard, the underlying conversation history is untouched, and you can turn it off at any time.
- **Background one-shot model calls unified onto cross-model fallback and usage accounting**: around 15 background calls that "use model capability but don't need a full Agent" — Recap, Dreaming, Knowledge Space compilation, skill auto-review, the Hooks `prompt` handler, session titling, awareness extraction, image OCR, Knowledge Space maintenance, Sprite, the notes toolkit, and more — now go through a real cross-model automatic fallback (instead of picking a model once at construction time and never changing) and are billed under precise purpose tags. A new global default model chain applies unless a feature overrides it. Dashboard usage trends gained a matching two-level drill-down by feature category / precise purpose tag.
- **Knowledge Space adds automatic OCR archiving for scanned PDFs**: importing a scanned PDF with no text layer now automatically rasterizes it page by page and runs vision recognition; import returns immediately while processing happens in the background, and a failed page can be retried individually without re-importing the whole document.
- **Knowledge Space binding and bulk import get real progress feedback**: binding an external knowledge space or bulk-importing material now shows real per-file progress (previously even "rebuild index" only counted by knowledge space, which was effectively no progress at all for a single-space scenario). New empty-state variants distinguish "scanning / failed / genuinely empty / nothing selected", plus a unified Knowledge Space activity panel, a sidebar busy / just-synced indicator, and desktop notifications when a scan or import finishes.
- **Fixed an occasional backend stall**: synchronous IO such as database and config reads/writes used to run on the async worker and could stall the whole tokio runtime; it no longer does. Also fixed a layout jitter on hover in the `ask_user` dialog, and added `Shift+Enter` for a newline in the chat input box.

## Conversation experience

- **Pasted long-text attachments are now editable in preview**: the preview card for an auto-generated pasted-text attachment now supports editing the content directly and saving (which replaces the pending file and refreshes the character/line counts), plus copy and cancel; the main chat, quick chat, and Knowledge Space chat all share this preview/edit experience.
- **Refined chat typography and incognito experience**: core chat text moved to a darker color; the title bar's quick-compact button was removed (compaction is still available from the status bar / context entry points); incognito copy is clearer, the Ghost icon uses a more neutral treatment, the no-title empty state now uses the main logo, and the composer gets a dark shell while incognito is on.
- **Dark theme shifted to a more neutral black-and-gray look**: the dark theme's core color moved from a blue-leaning slate to a neutral black-gray, with local dark tokens (like the incognito composer) debiased to match; the background animation is now off by default, reducing visual noise for new users on first launch.
- **Codex built-in model catalog adds GPT-5.6 Sol/Terra/Luna**: signing in via the ChatGPT/Codex OAuth subscription path now also surfaces the latest flagship models, with the default model upgraded to `gpt-5.6-sol`.

## Fixes & stability

- **Fixed file preview syntax highlighting**: opening a code file from workspace output, message file chips, diff jump links, or the shared file preview panel no longer falls back to plain-text rendering when language metadata was missing; recognition was also extended to more TypeScript variants and common extensionless code filenames.
- **Markdown file links reuse the unified file-type icon set**: local file hyperlinks now use the same colored file-type icons as attached files, instead of always showing a generic code icon.
- **Extension-driven browser control now surfaces the live BrowserPanel**: when a Chrome extension takes over a real browser tab, BrowserPanel frame capture now prioritizes the session's own extension-claimed tab, preventing another session's browser actions from popping open the current side panel.
- **Smoother long-session switching and sidebar reloads**: switching to an uncached session now switches the session shell immediately and loads history asynchronously; during startup/reload the sidebar waits for first-paint data to settle before rendering the list, reducing jank and flicker for long sessions and long session lists.
- **More accurate provider network-error detection**: transport-layer errors such as connection errors and DNS lookup failures are now classified as timeouts and trigger backoff retry instead of being misclassified as unknown errors; related notices now distinguish "rate limited", "upstream busy", and "network / proxy / DNS unreachable".
- **Clearer local Chrome extension install guidance**: the alpha/dev fallback in the install wizard now shows the local extension directory first, with a prompt to enable developer mode at `chrome://extensions` and drag that directory in.
