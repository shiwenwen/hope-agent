# Hope Agent 0.14.0 — File Diffs and Conversation Reading

> [简体中文](https://github.com/shiwenwen/hope-agent/blob/v0.14.0/docs/release-notes/v0.14.0.md) · **English**

> Released: 2026-06-29

> See [`CHANGELOG.md`](https://github.com/shiwenwen/hope-agent/blob/v0.14.0/CHANGELOG.md#0140---2026-06-29) for the full list.

The headline of v0.14.0 is a focused pass on **how file changes are presented and how long conversations read**: after a file tool succeeds you can expand the tool call inline to view a unified diff, while the right-hand diff panel gains professional capabilities like jump-to-change, split view, folding, search, and line-number preview; completed conversation turns now collapse their intermediate steps by default, and long content and attached files are more restrained, making long-running sessions easier to skim. This release also brings conversation input history and quick prompts, a smoother background-task panel, and the ability to hand long-running runtime timeouts back to a unified user policy — plus a unified, security-checked preview pipeline for external favicons.

## Highlights

- **Inline diffs for file changes**: after `write` / `edit` / `apply_patch` and other file-changing tools succeed, expanding the tool call in the conversation now shows a fixed-height, scrollable unified diff right there; multi-file patches are split per file, so you can verify what changed without switching to the side panel. The right-hand diff panel entry is still available.
- **A professional-grade diff panel**: viewing a tool-rewritten file now auto-positions to the first change and supports previous / next jumps, unified / split view toggle, folding unchanged context, ignoring whitespace changes, inline highlighting, multi-file search, scroll-position memory, and clicking a line number to preview the corresponding file line or copy a `path:line` location; multi-file diff headers are now sticky, and large diffs render in chunks to avoid stalling on too many lines at once.
- **Completed turns auto-collapse intermediate messages**: once a conversation turn ends, the preceding thinking, tool calls, and intermediate assistant messages collapse by default, leaving only a "how long / how many messages processed" summary and the final reply — cutting noise and DOM node count in long tasks. The summary expands on demand, and older messages in past sessions fold the same way. A toggle in Settings → "Conversation & Context" turns this off.
- **Conversation input history and quick prompts**: with an empty draft in the input box you can browse the current session's recent inputs with `↑` / `↓`; the toolbar at the bottom of a user message gains "Add to quick prompts", after which `#` in the input box brings up the quick-prompt menu for fast insertion, and the placeholder now hints at `@` / `#`.
- **Runtime timeouts are now user-policy driven**: runtime timeouts for long-running work (scheduled-task runs, MCP calls, ACP turns, async jobs, and more) are now governed by a unified user policy and can be set explicitly to unlimited (`0` = no timeout), so a short model-supplied timeout no longer cuts off work that is meant to run stably for a long time; a new "model runtime timeout override" policy lets you allow, audit-only, or ignore model-supplied timeout values. A positive user budget remains a hard cap — model overrides may only tighten it, never relax it.

## File changes and edit visualization

- **Clearer line counts for file edits**: the `+N/-M` line counts in the file-edit tool, the workspace, and the diff panel switch to an animated odometer, showing a safely inferred estimate while `edit` / `apply_patch` runs and then calibrating to the real diff result on completion; a file-type icon now precedes the target file name in the edit tool row, making it easier to scan which file is being changed. Failed or no-op edits no longer falsely show line changes (the real backend diff metadata is required on completion).
- **Cleaner targets in file tool calls**: `read` / `write` / `edit` / `apply_patch` and other file tools in the conversation no longer show the full path directly, switching to a file-type icon plus the file name; hovering the name still reveals the full path, and multi-file patches list target paths in the tooltip.
- **View workspace Git changes as a diff**: clicking the "Changes" row under the workspace "Environment" section now opens the right-hand diff panel, showing the Git changes inside the current session's working directory per file with per-file collapse / expand; in server mode this endpoint is also tightened to the session working directory, and symlinks are shown by their Git link text rather than followed to read files outside the workspace.

## Conversation reading

- **More restrained long content and attached files**: very long user messages collapse by default into a gradient-masked preview that expands to the full text; when an assistant reply generates or modifies many files, the first 6 are shown by default and the rest expand on demand. When a completed turn auto-collapses, files produced during tool steps are lifted to the bottom of the final reply rather than hidden inside the "processed" details.
- **Viewing a subagent conversation no longer navigates away from the main chat**: beyond the left conversation list, subagent conversation entry points in message bubbles, the team panel, the background-task panel, and the workspace now all open a live overlay that reuses the main conversation message stream; viewing a subagent from the message conversation page no longer switches away from the current main-Agent session.

## Background tasks

- **The background-task panel follows task state more smoothly**: triggering a background task auto-expands the right-hand panel; live output intelligently follows to the bottom while running, preserves position when you scroll back through history, and offers a jump-to-bottom control; each background-task entry can expand / collapse on its own so long output doesn't blow out the whole panel.
- **Background tasks are directly actionable from the workspace**: the background-task block in the workspace now reuses the same task-row capabilities as the background-task panel — expand to view live output / result previews, cancel still-running tasks, and share expansion state with the standalone panel; the workspace keeps its compact display and a "view all" entry.

## Other improvements

- **Responses / Codex tool rounds no longer repeat the same progress line**: tool-call rounds now write already-streamed assistant progress text back into the model-visible history so the next sampling after a tool can pick up where it left off; only the final reply is persisted on a natural finish, and a partial reply already seen is preserved when the user stops — avoiding lost or duplicated context downstream.
- **Fuller, faster sidebar search recall**: global session search now matches session titles and uses an FTS5 trigram index to fill in body substring matches (such as word fragments / common Chinese-English fragments) instead of relying on a `%LIKE%` scan over the whole message table; a title hit opens the session directly rather than jumping to a non-existent message bubble.
- **Backend-originated notifications follow the UI language**: the Rust backend can now read the app language preference for outbound text it sends directly — IM-channel startup recovery / session handover / tool approval / ask_user interactions, scheduled-task failure delivery wrappers, and the like — instead of being fixed to English; the key + fallback rendering boundary for frontend UI events is preserved so a model's language preference isn't mistaken for the product UI language.
- **More accurate progress-update settings copy**: the tool-call description toggle in Settings → "Conversation & Context" is reframed as "progress updates", explaining that it controls short progress on key steps and turn summaries; all 12 UI languages are updated in sync, and the model-specific recommendation to enable it has been removed.

## Security

- **External favicons go through a unified, secure preview pipeline**: Markdown web links, URL preview cards, and workspace sources can now show site favicons, but the icons are uniformly fetched by the backend through SSRF validation, per-hop redirect validation, size limits, and image-type validation before being converted to a data URL; a page-declared favicon is preferred, falling back to `/favicon.ico` on failure. Favicon fetching for Markdown links now triggers on hover / focus and is bounded by caching, concurrency, and count limits, avoiding unbounded backend requests when a history message renders many external links at once.
