# Hope Agent 0.22.0 — Embedded Terminal, Git/PR Panel, and LSP Diagnostics

> [简体中文](https://github.com/shiwenwen/hope-agent/blob/v0.22.0/docs/release-notes/v0.22.0.md) · **English**

> Release date: 2026-07-21

> See [`CHANGELOG.md`](https://github.com/shiwenwen/hope-agent/blob/v0.22.0/CHANGELOG.md#0220---2026-07-21) for details.

v0.22.0 rounds out the "do the whole coding loop inside the app" story: there's now a real interactive terminal docked below the conversation, the Git / PR panel closes the full loop from a local branch to pushing and opening a PR, and once a language server is configured the AI automatically sees your workspace's semantic diagnostics each turn so it catches type errors faster.

## Highlights

- **A new embedded terminal below the conversation**: a genuine interactive terminal is docked right under the chat, with multiple tabs, drag-to-resize height, maximize, and a `⌘/Ctrl+J` toggle. A new terminal automatically opens in the effective working directory of the current session or project Worktree, uses a high-contrast palette that follows the app's light / dark theme, and replays output within a bounded window. Desktop and HTTP / Web share the same core implementation — the remote terminal is off by default, guarded by the live `filesystem.allowRemoteWrites` permission gate, and revoking that permission immediately terminates any remotely created shell.
- **The Git / PR panel now closes the full interaction loop**: whether you're on a detached HEAD, a local-only branch that hasn't been pushed, a pushed branch with no PR yet, an existing PR, or GitHub is temporarily unavailable, the panel always shows a clear, actionable state instead of leaving you to guess the next step at an empty panel. A local-only branch can be pushed — or "pushed and turned into a PR" — in one click, and on success the in-app PR panel opens directly; when PR detail sync is delayed or fails, retry and an open-on-GitHub fallback remain available.
- **The AI can now see LSP semantic diagnostics**: with a language server configured (rust-analyzer / tsserver / pyright, etc.), the AI automatically receives your workspace's semantic diagnostics each turn — files you just edited this turn first — helping it catch type errors and warnings faster and saving a round trip.

## Changed

- **The apt / dnf software repository moved to a new address, `https://repo.hopeagent.ai`**: in v0.21.0 the installers crossed GitHub's hard 100 MB single-file limit across the board, structurally stalling the old GitHub Pages repository. The apt / dnf repository is now hosted on Cloudflare R2 — no single-file limit and free egress, a more natural home for a frequently downloaded package repo. **If you're still using the old repo (`shiwenwen.github.io/hope-agent-linux-repo`), you need to reconfigure the source address once per the README** to keep receiving updates; downloading `.deb` / `.rpm` / AppImage directly, or using the desktop app's self-update, is unaffected.

## Fixes & stability

- **The Workbench "Files" section no longer mislabels tool-produced files as "read"**: a file that was read first and later produced by a tool (image generation / attachment / exec), once it scrolls out of the current message window in a long session, now correctly shows as "output" and sorts to the front by production time — consistent with how it appears live within the window.
- **The Git / PR panel no longer spins forever**: fixed a persistent loading state caused by an empty request key hitting `null === null`; network requests now fire only when the prerequisites are met, and PR discovery and the periodic refresh of an existing PR run on their own separate cadences.
