# Hope Agent 0.17.0 — 视觉桥与后台模型统一降级

> **简体中文** · [English](https://github.com/shiwenwen/hope-agent/blob/v0.17.0/docs/release-notes/v0.17.0.en.md)

> 发布日期：2026-07-10

> 详见 [`CHANGELOG.md`](https://github.com/shiwenwen/hope-agent/blob/v0.17.0/CHANGELOG.md#0170---2026-07-10)。

v0.17.0 的主线是**视觉桥（Vision Bridge）**：当主对话模型不支持视觉时，不再简单丢图，而是用一个独立配置的视觉模型把图片转成文字再注入对话。围绕这条主线，本版还把 Recap、Dreaming、知识空间编译等约 15 个后台模型调用统一接入了跨模型自动降级与用量记账，知识空间新增扫描版 PDF 自动 OCR、绑定与批量导入的真实进度反馈，并修复了一处会拖垮后端的同步 IO 阻塞问题。

## 亮点

- **视觉桥（Vision Bridge）**：主对话模型不支持视觉（如纯文本的 DeepSeek 等）却收到图片时，可配置一个独立的视觉转述模型把图片转成文字再注入主对话，替代此前「丢图 + 占位符」的降级方式。默认关闭，仅显式配置视觉模型后启用；同时覆盖用户上传图片与浏览器截图 / 生图结果 / PDF 页面等工具产生的图片；用量单独记入 Dashboard「视觉转述」类别，会话历史本身不受影响，随时可关闭复原。
- **后台一次性模型调用统一接入跨模型降级与用量记账**：Recap、Dreaming、知识空间编译、技能自动审阅、Hooks `prompt` 处理器、会话标题、Awareness 提取、图片 OCR、知识空间维护、Sprite、笔记三件套等约 15 个「用到模型能力但不需要完整 Agent」的后台调用，现在统一接入真正的跨模型自动降级（而非构造期选中即不再变化），并按精确用途标签统一记账；新增全局默认模型链，各功能仍可分别覆盖。Dashboard 用量趋势相应新增按功能大类 / 精确用途标签两级下钻。
- **知识空间新增扫描版 PDF 自动 OCR 归档**：导入没有文字层的扫描版 PDF 时会自动逐页光栅化并做视觉识别，导入立即返回、后台异步处理，单页识别失败可单独重试而无需重新导入整份文档。
- **知识空间绑定与批量导入补齐真实进度反馈**：绑定外部知识空间、批量导入资料现在有按文件的实时进度（此前「重建索引」的进度也只按知识空间计数，单空间场景等同无进度）；新增区分「扫描中 / 失败 / 真空 / 未选中」的空状态提示、统一的知识空间活动面板、侧边栏忙碌 / 刚同步指示器，以及扫描 / 导入完成后的桌面通知。
- **后端偶发僵死修复**：数据库与配置读写等同步 IO 此前会跑在异步 worker 上拖垮整个 tokio runtime，现已移出异步路径；同时修复了 `ask_user` 弹窗悬停时的布局抖动，并新增聊天输入框 `Shift+Enter` 换行。

## 对话体验

- **长文本粘贴附件支持预览编辑**：粘贴生成的文本附件预览卡片现在支持直接编辑内容后保存（自动替换待发送文件并刷新字符数 / 行数统计），也支持复制与取消；主对话、快速对话与知识空间对话共用同一套预览与编辑体验。
- **聊天排版与无痕对话体验优化**：核心聊天文字改为更深的配色；标题栏移除快捷压缩按钮（压缩功能仍可从状态栏 / 上下文入口进入）；无痕对话文案更清晰，Ghost 图标改为更中性的处理，无标题时的空状态改用主 Logo，开启无痕后输入框整体呈深色底。
- **暗色界面改为更中性的黑灰观感**：暗色主题核心色从偏蓝的 slate 调整为中性黑灰，输入框无痕态等局部暗色 token 同步去蓝相；背景动画默认关闭，减少新用户启动后的视觉干扰。
- **Codex 内置模型目录新增 GPT-5.6 Sol/Terra/Luna**：使用 ChatGPT/Codex OAuth 订阅登录时也能选到最新旗舰模型，默认模型升级为 `gpt-5.6-sol`。

## 修复与稳定性

- **文件预览语法高亮修复**：从工作台输出、消息文件卡片、diff 跳转链接与共享文件预览面板打开代码文件时，不再因缺少语言信息而回退成纯文本渲染；新增对更多 TypeScript 变体与常见无后缀代码文件名的识别。
- **Markdown 文件链接复用统一文件类型图标**：本地文件超链接现在使用与下挂文件一致的彩色文件类型图标，不再统一显示通用代码图标。
- **插件浏览器控制会唤出实时 BrowserPanel**：通过 Chrome 扩展接管真实浏览器标签页时，BrowserPanel 截帧改为按会话优先捕获 extension claimed tab，避免其它会话的浏览器动作误打开当前右侧面板。
- **切换长会话与重载侧边栏更顺滑**：切换未缓存会话时先立即切换会话 shell、异步加载历史消息；启动 / reload 期间侧边栏会等首屏数据稳定后再渲染列表，减少长会话和会话列表的卡顿与重影。
- **Provider 网络错误识别与提示更准确**：连接错误 / DNS 解析失败等传输层错误会归类为超时并触发退避重试，不再误判为未知错误；相关提示会区分「限流」「上游繁忙」「网络 / 代理 / DNS 不可达」。
- **Chrome 扩展本地安装引导更清楚**：安装向导的 alpha / dev fallback 改为先显示本地扩展目录，并提示在 `chrome://extensions` 开启开发者模式后拖入该目录。
