# Hope Agent 0.12.0 — Chrome 扩展浏览器后端

> **简体中文** · [English](https://github.com/shiwenwen/hope-agent/blob/v0.12.0/docs/release-notes/v0.12.0.en.md)

> 发布日期：2026-06-23

> 详见 [`CHANGELOG.md`](https://github.com/shiwenwen/hope-agent/blob/v0.12.0/CHANGELOG.md#0120---2026-06-23)。

v0.12.0 的主线是新增可选的 **Chrome 扩展浏览器后端**：通过 MV3 扩展 + 原生消息桥驱动你**已登录的真实 Chrome**（含全部 Cookie 与会话）完成网页操作，适合需要登录态的任务；扩展不可用、且动作不依赖真实 Chrome 状态时，自动降级到现有的隔离 CDP 后端。桌面安装包已随附扩展文件，商店上架前即可本地一键安装。此外本版还带来压缩后可检索回看历史会话，以及上下文压缩反馈、设置数字输入等体验修复。

## 亮点

- **Chrome 扩展浏览器后端**：可选通过 MV3 扩展 + 原生消息桥驱动你已登录的真实 Chrome（含全部 Cookie 与会话）完成网页操作，适合需要登录态的任务。8 个浏览器动作（navigate / snapshot / act / observe / control / tabs / profile / status）经扩展驱动真实标签页，真实页面经 claim 接管；raw CDP 高级逃生舱默认开启但**每次调用都需审批、永无「始终允许」**，读取 Cookie 等危险方法已拦截。扩展不可用且动作不依赖真实 Chrome 状态时，自动降级到现有隔离 CDP 后端；可在「设置 → 浏览器 → 高级」关闭 raw CDP 或切换后端偏好（扩展优先 / 仅隔离 CDP / 仅扩展）。
- **桌面版自带扩展、可直接本地安装**：商店上架前，桌面安装包已随附扩展运行时文件，可在「设置 → 浏览器」一键打开扩展目录并在 Chrome「加载已解压的扩展程序」装上，无需自行下载源码。扩展工作目录稳定在 `~/.hope-agent/extension/browser/`，App 更新 / 移动不致已加载扩展失效；原生消息 host 在桌面启动时自注册并以 broker token 鉴权，配合扩展退避重连，App 重启后自动恢复、无需手动重载扩展。当扩展为默认但不可用、静默回退 CDP 时，聊天顶部会弹出可关闭的引导横幅，连接成功后给出回执。
- **压缩后可检索回看历史会话**：新增模型内置工具 `sessions_search`，可在上下文压缩后按关键词回查当前或历史普通会话的持久化消息，并返回命中附近的上下文窗口——让压缩后的上下文像「软记忆」，需要时仍能精确取回历史对话细节，而不必手写 SQL。全局搜索默认只覆盖普通非无痕会话，避免混入 cron / subagent / IM channel / 知识空间侧栏会话。同时新增 `ha-data-stores` 内置技能，向模型说明 Hope Agent 本地数据根目录、主要 SQLite / 文件存储与只读查询规范，便于排障时优先用产品工具、必要时再安全地查询底层数据。

## 其它改进

- **任务列表与上下文用量条更清爽**：消息里的任务清单、工作台任务进度在所有任务完成后默认折叠，仍可手动展开查看明细；聊天输入框底部的 token 用量进度条改为内缩胶囊样式，不再贴着圆角边缘溢出。
- **手动上下文压缩状态反馈更完整**：`/compact`、标题栏、工作台与 `/context` 卡片触发的压缩现在共用同一套进度 / 完成 / 失败系统消息，并在刷新或重新打开会话后保留最终结果与压缩后的 token 用量；摘要生成超时时会明确区分「已清理但摘要超时」与「摘要超时未压缩」。
- **设置里的数字输入编辑体验修复**：设置页数字输入改为失焦时再提交，避免输入中途被范围钳制导致无法删除旧数字；天气经纬度这类可空字段现在也能通过清空输入恢复为未设置状态。
